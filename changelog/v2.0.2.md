# 2.0.2 — In Development

Minecraft 1.21.1 | Fabric, NeoForge

## Internal

| Tag | Change |
|---:|---|
| ![improvement] | <details><summary>Inter-server (master↔slave) handlers no longer log routine disconnects as ERROR</summary><br>The multi-server master TCP listener logged a full ERROR + stack trace whenever a peer dropped before/without completing the handshake (e.g. a port scan, or a slave crashing mid-handshake) — appearing in server logs even with no players online. Both ends now classify exceptions by *expectedness* rather than connection phase:<br><br><ul><li>**Master** — routine `IOException` (e.g. "Connection reset") on an un-handshaked connection logs at WARN as an aborted connection attempt; on an established slave it logs at DEBUG (the disconnect is already reported at INFO by `removeChildConnection`, avoiding a double log). Any non-routine cause — including a decode/runtime fault *before* handshake — keeps the full ERROR + stack trace so genuine protocol bugs are not hidden.</li><li>**Slave** — routine `IOException` from the master now logs at DEBUG (the reconnect is already reported by `channelInactive`); only genuine failures remain ERROR + stack trace. Previously every master disconnect spammed ERROR with a trace.</li></ul>The peer's remote address is captured in `channelActive` and included in master-side log lines (replacing the uninformative `'null'`), so the source of a connection attempt is identifiable.</details> |
