# v2.0.0_ALPHA Changelog

**Minecraft Version:** 1.21.1  
**Platforms:** Fabric, NeoForge, Quilt  
**Status:** In Development

---

## Summary

Major documentation overhaul, comprehensive code analysis with 68 verified issues, and migration of the in-game test suite framework from BankSystem to ModUtilities as a shared library.

---

## New Features

### In-Game Test Suite Framework
- Migrated test framework from BankSystem to ModUtilities as shared library
- `TestCategory` redesigned from enum to extensible class — any mod can register its own categories
- `TestRunner` now accepts configurable mod name prefix
- `TestCommandRegistration` provides reusable `/modname test` command wiring
- `TestRegistry` with `ENABLE_TESTS` kill switch for production builds
- `TestSuite` abstract base with 16 assertion helpers (assertEquals, assertTrue, assertFalse, assertNotNull, assertNull, assertThrows — each with message overloads)
- 7 internal `mu_` prefixed test categories (networking, streaming, settings, persistence, events, multi_server, gui)
- 80 built-in tests across 5 test suites:
  - `EventTests` (10 tests) — Signal + DataEvent lifecycle
  - `SettingsTests` (21 tests) — Setting, SettingsGroup, SettingsStore JSON roundtrip, NBTJsonParser
  - `PersistenceTests` (18 tests) — TimeInterval logic, DataArchiveChunk save/load, NBTFileParser
  - `GuiLogicTests` (17 tests) — Point rotation, Plot.Axis math, LayoutGrid, TextBox regex, Rectangle geometry
  - `NetworkingTests` (14 tests) — RequestRegistry, StreamRegistry, MultiServerConfig, concurrent access
- Tests verify 8 known issues: #4, #8, #24, #25, #27, #29, #43, and #1 (by inspection)

### Commands
- `/modutilities test` — Run all registered test suites
- `/modutilities test list` — List available test categories
- `/modutilities test <category>` — Run specific category

## Bug Fixes

### GUI Geometry Fix
- `Point.getRotated` — fixed `(int)` truncation causing incorrect rotation results; now uses `Math.round()` for proper rounding of floating-point coordinates

### Critical GUI Fixes (Task #13)
- `Gui.setGuiScale` — fixed parameter shadowing that stored invalid value before validation, causing divide-by-zero in scissor/mouse/render scaling
- `GuiScreen.isInitialized` — `isInitialized()` now correctly returns true after init
- `GuiContainerScreen.isInitialized` — same fix for container screen variant
- `TabElement.setSelectOutlineThickness` — fixed `Math.min(0, thickness)` → `Math.max(0, thickness)` so outline can render
- `TabElement.removeTab` — now correctly removes the removed tab's title element instead of the selected tab's

### ModChecker Platform Fixes (Task #14)
- Created NeoForge `ModCheckerImpl` — `@ExpectPlatform` no longer fails with `AssertionError`
- Moved Fabric `ModCheckerImpl` to correct package (`net.kroia.modutilities.fabric`)

### Data/Persistence Fixes (Task #15)
- `DataArchiveChunk.loadInternal()` — removed duplicate `load()` call that caused data corruption
- `TimeInterval.overlapsWith()` — uses `getEndTime()` getter instead of raw field; active intervals now correctly report overlap

### Server-Side Crash Fixes (Task #16)
- `UtilitiesPlatform.getRegistryAccess()` — guards client-side call with `isClient()` check; no longer crashes dedicated servers
- `isServerInitialized()` / `isClientInitialized()` — return `false` instead of throwing `IllegalStateException`
- Added `@Environment(EnvType.CLIENT)` to `SandboxClientHooks` and `TestScreen`
- Added `UtilitiesPlatform.isPlatformSet()` for safe platform availability checks

### ByteBuf and Buffer Handling Fixes (Task #17)
- `ExtraCodecUtils` — `FRIENDLY_BYTE_BUF_CODEC` and `REGISTRY_FRIENDLY_BYTE_BUF_CODEC` now safely extract readable bytes instead of calling `array()` on potentially direct buffers
- `MultiServerPacketRegistry.createForwardPacketPayload` — fixed `array()` usage and added buffer release
- `MasterPacketHandler` / `SlavePacketHandler` — added try-finally to release `RegistryFriendlyByteBuf` after handling
- `GenericRequestPacket` / `GenericResponsePacket` — added buffer release in all handler methods

---

### High-Priority Tasks

#### Networking Reliability (Task #18)
- `GenericRequest.decodeHandleEncodeOnServer/Master/Slave` — futures now complete exceptionally on handler errors instead of leaking; wrapped in try-catch and `whenComplete` instead of `thenAccept`
- `RequestManager` — added `cleanupExpiredRequests(maxAgeMs)` method and `creationTimeMs` to request holders to prevent unbounded memory growth from un-responded requests

#### Streaming NPEs (Task #19)
- `StreamManager.handlePacketOnServer` — null-check `streamData` before accessing `playerUUID`
- `StreamSystem.handlePacket(StreamStartPacket)` — null-check stream registration; logs warning instead of crashing on mod version mismatch

#### Persistence File Handling (Task #20)
- `DataPersistence.getAbsoluteSavePath()` — clear error when `levelSavePath` not set instead of NPE
- `getJsonFiles`, `getFiles`, `getFoldes`, and chunk deletion — wrapped `Files.list()` in try-with-resources to prevent file handle leaks (was blocking directory operations on Windows)

#### High-Priority GUI Bugs (Task #21)
- `Gui.playLocalSound` — null-check player/level (no more NPE on disconnect)
- `TextBox` paste — cursor now advances by clipboard length after paste
- `LayoutGrid` — fixed redundant `+1` in row/column calculation; uses `Math.ceil()` alone
- `ContainerView.renderSlotHighlight` — fixed completely misaligned `drawGradient` arguments (was passing x+16 as z, color as height, blitOffset as colorTo)

#### Build Configuration (Task #22)
- `gradle.properties` — removed `quilt` from `enabled_platforms` since `settings.gradle` excludes the quilt subproject

#### Sandbox Fixes (Task #23)
- `SandboxOpenGuiPacket.handleOnClient` — now uses `SandboxClientHooks.openTestScreen()` to dispatch screen opening to the render thread
- `SandboxDataArchiveManager.loadAndSaveInternal` — added bounds check before `chunks.get(5)`
- `Sandbox.loadItemToHand` — uses `MAIN_HAND` instead of `getUsedItemHand()`

#### Settings System (Task #24)
- `SettingsStore.fromJson(List)` — missing groups silently skipped; old saves load gracefully when mod adds new groups
- `SettingsStore.fromJson(SettingsGroup)` — removed redundant dead null check
- `SettingsStore.setValueWithCast` — converts `Number` to target type (Integer/Long/Float/etc.) preventing `ClassCastException` from Gson's default Double deserialization

#### Thread Safety (Task #25)
- `RequestRegistry`, `StreamRegistry`, `MultiServerPacketRegistry.registry` — `HashMap` → `ConcurrentHashMap`
- `MultiServerPacketRegistry.registryAccess`, `MultiServerManager.instance` — added `volatile`
- `Signal`, `DataEvent` — `ArrayList` → `CopyOnWriteArrayList`; `notifyListeners()` iterates a snapshot and operates by listener identity to avoid concurrent modification
