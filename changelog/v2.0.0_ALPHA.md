# v2.0.0_ALPHA Changelog

**Minecraft Version:** 1.21.1  
**Platforms:** Fabric, NeoForge, Quilt

## Summary

Major documentation overhaul, comprehensive code analysis with 68 verified issues, and migration of the in-game test suite framework from BankSystem to ModUtilities as a shared library.

---

## New Features

### In-Game Test Suite Framework
- Migrated test framework from BankSystem to ModUtilities as shared library
- `TestCategory` redesigned from enum to extensible class — any mod can register its own categories
- `TestRunner` now accepts configurable mod name prefix
- `TestCommandRegistration` provides reusable `/modname test` command wiring
- `TestRegistry` with `ENABLE_TESTS` kill switch for production builds
- `TestSuite` abstract base with 16 assertion helpers (assertEquals, assertTrue, assertFalse, assertNotNull, assertNull, assertThrows — each with message overloads)
- 7 internal `mu_` prefixed test categories (networking, streaming, settings, persistence, events, multi_server, gui)
- 80 built-in tests across 5 test suites:
  - `EventTests` (10 tests) — Signal + DataEvent lifecycle
  - `SettingsTests` (21 tests) — Setting, SettingsGroup, SettingsStore JSON roundtrip, NBTJsonParser
  - `PersistenceTests` (18 tests) — TimeInterval logic, DataArchiveChunk save/load, NBTFileParser
  - `GuiLogicTests` (17 tests) — Point rotation, Plot.Axis math, LayoutGrid, TextBox regex, Rectangle geometry
  - `NetworkingTests` (14 tests) — RequestRegistry, StreamRegistry, MultiServerConfig, concurrent access
- Tests verify 8 known issues: #4, #8, #24, #25, #27, #29, #43, and #1 (by inspection)

### Commands
- `/modutilities test` — Run all registered test suites
- `/modutilities test list` — List available test categories
- `/modutilities test <category>` — Run specific category
