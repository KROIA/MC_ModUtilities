# 2.0.4 — In Development

Minecraft 1.21.1 | Fabric, NeoForge

## Security

| Tag | Change |
|---:|---|
| ![hardening] ![breaking] | <details open><summary>Forwarded multi-server packets now derive the sender identity from the authenticated connection instead of the packet</summary><br>Previously each forwarded packet carried a self-declared `senderServerID` string on the wire. An authenticated slave could set it to any value (e.g. an empty string or another slave's id) to impersonate a different origin and bypass `slaveID`-based trust gates in dependent mods.<br><br>**Fix:** the `senderServerID` field is removed from the `FORWARD_PACKET` wire entirely. The slave sends its id only during the handshake; the master reconstructs the sender identity from the authenticated per-socket handshake id, and the slave stamps the constant `"master"` for its sole authenticated upstream. Spoofing is now structurally impossible — there is no field to forge.<br><br>**⚠️ Breaking wire-format change:** the `FORWARD_PACKET` byte layout changed. A master and slave running **different** ModUtilities versions will mis-decode forwarded packets — the master and every slave MUST run the same ModUtilities version. The source API (`ForwardPacketContext`) is unchanged; only the wire format changed. (#13)</details> |

## API

| Tag | Change |
|---:|---|
| ![feature] | `GenericStream` now exposes `getSlaveServerID()` / `setSlaveServerID(...)`, giving server-sender streams the originating slave id (or `null` for a master-local stream) so subclasses can make per-origin decisions — parity with the request-side `slaveID` plumbing (FR-001) |
