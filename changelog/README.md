# Changelog Directory

This directory contains detailed version-specific changelogs for MC_ModUtilities.

## Structure

Each version has its own markdown file:
- `v2.0.0_ALPHA.md` - Version 2.0.0 Alpha changelog
- `v2.0.0.md` - Version 2.0.0 Release changelog
- `v2.1.0.md` - Future version changelogs
- etc.

## Naming Convention

- **Alpha/Beta versions:** `vX.Y.Z_ALPHA.md` or `vX.Y.Z_BETA.md`
- **Release versions:** `vX.Y.Z.md`
- **Hotfix versions:** `vX.Y.Z.1.md`, `vX.Y.Z.2.md`, etc.

## Changelog Generation

Changelogs are generated at the end of each version development cycle by:
1. Reviewing all completed tasks from COMPLETED_TASKS.md
2. Summarizing features, improvements, and fixes
3. Documenting API changes and breaking changes
4. Noting dependency updates
5. Listing known issues
