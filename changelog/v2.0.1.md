# 2.0.1 — In Development

Minecraft 1.21.1 | Fabric, NeoForge

## API

| Tag | Change |
|---:|---|
| ![feature] | <details><summary>DisplayBlock — render GUI elements on block faces in 3D world space</summary><br>The GUI library core (`Gui`, `GuiElement`, all element subclasses) is now common code that runs on both server and client. GUI elements render on block faces via offscreen framebuffer — the same `Label`, `Button`, `Slider`, `Plot`, etc. that work on screens now render identically on blocks.<br><br><ul><li>`IGraphics` interface extracted from `Graphics` — rendering abstraction with no-op defaults for server-side use</li><li>`IInputProvider` + `InputConstants` — GLFW-free input abstraction for common code</li><li>`ClientGraphics` — screen rendering backend (wraps `GuiGraphics`)</li><li>`WorldGraphics` — world-space rendering backend (direct vertex consumer)</li><li>`DisplayDemoBlock` + `DisplayDemoBlockEntity` — proof-of-concept block with animated sine/cosine plot</li><li>Framebuffer-based renderer with configurable resolution and render scale (supersampling)</li></ul></details> |
| ![feature] | <details><summary>Display Block API — reusable base classes for custom in-world display blocks</summary><br>Dependent mods can create custom display blocks by extending two classes and registering. No custom renderer needed.<br><br><ul><li>`AbstractDisplayBlock` — base block with facing, interaction handling, and group recalculation</li><li>`AbstractDisplayBlockEntity` — base block entity with grouping, sync, interaction, state transfer, and channel IDs</li><li>`AbstractDisplayBlockEntityRenderer` — generic config-driven offscreen renderer (replaces per-type renderers)</li><li>`DisplayInteractionScreen` — generic interaction screen with `GuiStateSync`-based state sync</li><li>`DisplayConfig` — configuration record (virtualWidth/Height, renderScale, faceOffset, ShapeProvider)</li><li>`ContentBuilder` — functional interface for GUI content creation</li><li>`ShapeProvider` — functional interface for per-facing voxel shapes</li><li>`DisplayClientHooks` — client-only hooks for opening interaction screens</li><li>Channel ID support prevents blocks with different channel IDs from merging into groups</li></ul></details> |
| ![feature] | <details><summary>Packet-based display input sync — interaction screen works on dedicated servers</summary><br>`DisplayInputSyncPacket` (C2S) carries serialized GUI input state (sliders, textboxes, checkboxes, button clicks) plus a closing flag. Server handler applies input via `GuiInputSerializer`, calls `onInputSynced()`, syncs to clients, and releases the editor lock on close.<br><br><ul><li>`GuiInputSerializer` — serializes/deserializes input element state to/from `CompoundTag` by element index path</li><li>`DisplayInputSyncPacket` — C2S packet with `BlockPos` + `CompoundTag` + closing flag</li><li>`DisplayNetworking` — registers C2S packet via Architectury `NetworkManager`</li></ul></details> |
| ![feature] | `EmptyButton.clickCount` + `syncClickCount()` — click counter enables button event propagation through `GuiStateSync` |
| ![improvement] | `GuiStateSync` — syncs `EmptyButton` click count between server and client GUI trees |
| ![improvement] | `Button.serializeState()`/`deserializeState()` now includes label text |
| ![improvement] | `CheckBox` — added `setText()`/`getText()` methods; label text included in state serialization |
| ![breaking] | <details><summary>GUI library package restructure — client-only classes moved to `gui/client/`</summary><br>`GuiScreen`, `GuiContainerScreen`, `ContainerView`, `InventoryView`, `ClientGraphics`, `ClientInputProvider` moved to `net.kroia.modutilities.gui.client` package. The old `Graphics` class is renamed to `ClientGraphics`. Import paths must be updated in dependent mods.<br><br>`Gui` and `GuiElement` no longer have `@Environment(EnvType.CLIENT)` — they are common code now.</details> |

## UI / UX

| Tag | Change |
|---:|---|
| ![feature] | DisplayBlock sandbox demo: `/modutilities giveDisplayBlock` places a block rendering a live signal dashboard with animated sine/cosine plot, identical to `/modutilities openExample dashboard` |
| ![feature] | Display Panel sandbox demo: `/modutilities giveDisplayPanel` gives a thin glass-pane-style display block with click counter, checkbox, and text input |
| ![feature] | Chart Demo sandbox block: `/modutilities giveChartDemo` places a display block with interactive line chart (sine/cosine/square waves, pan/zoom) — tests scissor clipping on block face |
| ![improvement] | Display blocks no longer show the block selection outline wireframe — the outline was overlapping the rendered display content |
| ![improvement] | `ClientGraphics.setEnableShadow(boolean)` — toggle text drop shadows per graphics backend |
| ![bugfix] | <details><summary>TextBox key events propagated to screen, closing GUI on "E" press</summary><br>Pressing "E" while typing in a focused TextBox closed the screen because the key event escaped to GuiScreen's inventory-close handler.<br><br><ul><li>Moved the `GLFW_KEY_E` close logic inside the `getFocusedElement() == null` guard in `GuiScreen.keyPressed()`</li><li>Changed `TextBox.keyPressed()` to return `true` for all keys when focused, preventing event propagation</li></ul></details> |
| ![bugfix] | CloseButton `setOutlineColor()` NPE when called before lines are initialized |
| ![bugfix] | <details><summary>Scissor clipping incorrect on display block rendering</summary><br>`renderGuiToTexture()` never activated offscreen scissor mode — elements calling `enableScissor()` used screen-space coordinates (`GuiGraphics.enableScissor`) instead of framebuffer-space coordinates (`GL11.glScissor`). Fixed by calling `gui.setOffscreenScissorMode(scale, texH)` before and `gui.clearOffscreenScissorMode()` after the GUI render pass. Also fixed `scissorPause()` to use raw GL disable in offscreen mode.</details> |
| ![bugfix] | <details><summary>Clouds and translucent blocks flickered when a display block was in front of the player</summary><br>The offscreen framebuffer render pass in the display block renderer modified shader color and fog start/end without restoring them, causing subsequent cloud and translucent rendering to use incorrect values.<br><br><ul><li>Added shader color save/restore around the GUI render pass</li><li>Added fog start/end save/restore (was only saving fog color before)</li><li>Removed redundant `RenderSystem.bindTexture` call</li></ul></details> |
